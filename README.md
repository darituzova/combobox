## Инфраструктурный слой
- Брокер сообщений: Развернут MQTT-брокер Eclipse Mosquitto в среде Docker. Настроен прием подключений на стандартном порту (1883) с конфигурацией, допускающей подключение локальных клиентов без авторизации. Это обеспечивает беспрепятственную отладку C++ прошивок на этапе прототипирования.

- База данных: Интегрирована СУБД PostgreSQL с профильным расширением TimescaleDB, оптимизированным для работы с временными рядами. Разработан скрипт инициализации, создающий структуру хранения данных и конвертирующий базовую таблицу в гипертаблицу. Данное решение гарантирует высокую производительность при параллельной записи метрик от множества станков.

## Программная архитектура бэкенда
- Основа приложения: Серверная часть реализована на языке Python с использованием асинхронных фреймворков FastAPI (для HTTP) и FastStream (для MQTT).

- Чистая архитектура: Проект структурирован с соблюдением принципов Clean Architecture. Использован IoC-контейнер Dishka для управления зависимостями, что позволило изолировать бизнес-логику от фреймворков и базы данных.

- Валидация трафика: Внедрены строгие Pydantic-схемы. Входящий JSON-трафик автоматически проверяется на соответствие утвержденной структуре. Пакеты с нарушенной структурой или неверными типами данных отсекаются до этапа обработки.

- Управление временем: В слое бизнес-логики (интеракторах) реализован механизм автоматической генерации и присвоения метки времени (UTC) каждому входящему пакету. Это снимает нагрузку с конечных устройств и исключает рассинхронизацию времени в системе.

- Слой доступа к данным (Gateway): Разработан асинхронный шлюз на базе SQLAlchemy для трансляции доменных сущностей телеметрии в SQL-запросы и их последующего сохранения в БД.

## Тестирование и отладка
- Эмуляция устройств: Для проведения независимых испытаний написан скрипт-эмулятор. Он формирует тестовые пакеты, содержащие идентификатор узла и набор физических метрик (температура, вибрация, влажность, давление, энергопотребление), и публикует их в топик брокера.

- Сквозная проверка: Выполнено интеграционное тестирование всего конвейера. Зафиксировано успешное прохождение сигнала от публикации в MQTT до персистентного сохранения в TimescaleDB.

- Вспомогательный инструментарий: Для упрощения контроля со стороны разработчиков написан диагностический HTTP-эндпоинт, позволяющий сделать прямую выборку последних сохраненных записей из базы данных.

# Инструкция: Как запустить IIoT Backend на Windows

Бэкенд полностью упакован в Docker. Внутри автоматически поднимется сервер (FastAPI), база данных временных рядов (TimescaleDB), MQTT-брокер (Mosquitto) и сразу загрузятся исторические данные станков для тестирования.

#### Шаг 1: Установка Docker Desktop (Единственная необходимая программа)

1. Скачай и установи **Docker Desktop** для Windows с официального сайта: `[https://www.docker.com/products/docker-desktop/](https://www.docker.com/products/docker-desktop/)`
2. Во время установки убедись, что стоит галочка **«Use WSL 2 instead of Hyper-V»** (это современный движок для Docker на Windows).
3. Запусти Docker Desktop после установки (в трее рядом с часами должен появиться значок кита 🐳, и он должен гореть зеленым — _Engine running_).

#### Шаг 2: Получение файлов проекта

1. Скачай код проекта (через `git clone <ссылка на репозиторий>` или просто скачав ZIP-архив и распаковав его в удобную папку).
2. Убедись, что в корневой папке проекта (там же, где лежит файл `docker-compose.yaml`) находится файл с датасетом **`smart_manufacturing_data.csv`**. Без него исторические данные не загрузятся!
3. В этой же папке создай текстовый файл с названием **`.env`** (просто точка энв, без расширения .txt) и вставь туда настройки базы данных:
    ``` Plaintext
    DB_USER=admin
    DB_PASSWORD=secretpassword
    DB_HOST=postgres
    DB_PORT=5432
    DB_NAME=iiot_factory
    ```
    
    _(Обратите внимание: `DB_HOST=postgres` и порт `5432` — это правильные внутренние настройки для работы внутри Docker)._

#### Шаг 3: Запуск

1. Открой любую консоль (PowerShell, CMD или терминал в VS Code).
2. Перейди в папку с проектом:
    ``` Bash
    cd путь\к\папке\с\проектом
    ```
3. Выполни команду запуска:
    ``` Bash
    docker-compose up --build -d
    ```
    
    _Первый запуск займет пару минут (будут скачиваться образы базы данных и питона, собираться библиотеки). При последующих запусках сервер будет стартовать за секунду._
    

#### Шаг 4: Проверка работоспособности

Убедиться, что всё работает, очень просто:

1. Введи в консоли команду `docker-compose logs backend`. Если в конце написано `Application startup complete` — сервер жив!
2. Открой браузер и перейди по ссылке: 👉 **[http://127.0.0.1:8000/docs](https://www.google.com/search?q=http://127.0.0.1:8000/docs&authuser=4)** Ты должен увидеть интерактивную документацию Swagger со всеми доступными Контрактами (HTTP и WebSocket).

**Как остановить сервер?** Когда закончишь работу, просто введи в той же папке:


``` Bash
docker-compose down
```
